你现在是一个资深 C++ 桌面应用工程师，请从零开始为我开发一个 Ubuntu 桌面端应用。

# 项目目标

开发一个“晨间认知补给器”桌面应用：每天早上自动刷新 10 条内容，内容分为以下四类：
1. 思维/认知启发
2. 可培养的好习惯
3. 要学习的知识点
4. 从新闻中挖掘出的热点/趋势

这个应用不是简单资讯聚合，而是一个“结构化晨间内容组件”，目标是让用户每天用 3 分钟完成：
- 获取一个思考角度
- 认识一个可以培养的习惯
- 学到一个知识点
- 了解一个热点趋势
- 最终形成一个可执行动作

# 技术约束

建议技术栈如下（如有更合理替代方案，也可以自行选择，但必须保持纯 C++）：
- UI：Qt 6（优先）或 GTKmm（二选一，优先 Qt 6）
- 语言标准：C++20
- 构建系统：CMake
- 本地存储：SQLite
- 网络请求：Qt Network 或 libcurl
- JSON 解析：nlohmann/json
- 日志：spdlog（可选）
- 定时任务：Qt 的定时器或系统级调度
- 平台：Ubuntu Desktop 优先

# 产品形态

这是一个独立桌面应用，不是浏览器网页，不是 GNOME Shell 扩展，不是手机应用。

应用需要具备以下桌面体验：
- 启动后可驻留在系统托盘或以小窗形式常驻
- 支持开机自启
- 支持手动刷新和自动每日刷新
- 支持最小化到托盘
- 支持桌面低打扰展示
- 支持深色/浅色模式（至少预留接口）

# 核心功能需求

## 1. 今日内容面板
主界面展示 10 条内容，默认按以下配比：
- 思维/认知启发：3 条
- 好习惯：2 条
- 知识点：3 条
- 热点/趋势：2 条

每条内容卡片至少包含：
- 标题
- 类型标签
- 一句话摘要
- “为什么重要”
- “今日行动”建议
- 收藏按钮
- 跳过按钮
- 不感兴趣按钮

## 2. 内容生成与刷新
需要实现一个内容生成管线：
- 内容采集
- 内容清洗
- 去重
- 分类
- 打分
- 排序
- 输出今日 10 条内容

第一版可以允许用“内置种子内容 + 简单抓取内容”的方式实现，不要求一开始就接入复杂 AI 模型，但架构必须支持后续扩展。

## 3. 本地偏好与学习
需要记录用户行为：
- 点击了什么
- 收藏了什么
- 跳过了什么
- 不感兴趣了什么
- 停留时长（如果实现成本合理）

并根据这些行为调整后续推荐权重。

## 4. 设置页
至少要有以下设置项：
- 每日刷新时间
- 内容类型比例
- 新闻/热点来源开关
- 是否开机自启
- 是否最小化到托盘
- 外观模式
- 重置推荐偏好

## 5. 数据持久化
需要把以下数据持久化到 SQLite：
- 内容库
- 用户行为日志
- 用户偏好配置
- 每日生成的 feed
- 收藏列表

## 6. 日志与错误处理
需要有基本日志能力：
- 网络失败可重试
- 解析失败可降级
- 内容为空时使用本地兜底内容
- 启动异常要有明确日志

# 内容来源策略

第一版请用“可运行、可验证”的方式实现，不要卡在复杂的外部依赖上。

建议实现方式：
- 先内置一批高质量种子内容，保证产品可用
- 再接入 RSS 或公开新闻源作为可选扩展
- 对抓取内容做摘要和分类时，先用规则和关键词实现
- 架构预留 LLM 接口，但不要强依赖 LLM 才能运行
